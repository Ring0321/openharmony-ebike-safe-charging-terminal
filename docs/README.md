# Documentation

The `docs/zh` folder keeps the original Chinese project documents in a GitHub-friendly structure:

- `project-overview.md`: overall project introduction.
- `project-plan.md`: project plan and development goals.
- `terminal-development-plan.md`: complete terminal-side development plan.
- `detailed-product-development-plan.md`: product-oriented development plan.
- `hardware-wiring.md`: hardware wiring guidance.
- `four-relay-wiring.md`: K1-K4 relay function and wiring notes.
- `software-state-machine-and-demo.md`: current state machine and demo logic.
- `huawei-iotda-checklist.md`: Huawei Cloud IoTDA configuration checklist.
- `k1-k4-cloud-flashing-guide.md`: K1-K4 cloud firmware flashing notes.
- `current-development-status.md`: current verified development status.
- `reuse-summary.md`: reuse summary for similar competition projects.
- `su03t-safe-charge-binding.md`: SU-03T voice module binding notes.

Hardware diagrams are under `docs/hardware/drawio`, with an exported figure under `docs/hardware/images`.
