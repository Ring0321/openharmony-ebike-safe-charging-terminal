## 1. 本阶段目标

本阶段不是一次性接入所有硬件，而是先完成最关键的南向硬件动作：

```text
RK2206 GPIO
-> 继电器模块
-> 低压负载
-> 按键控制充电/断电
-> LCD、串口、MQTT 状态一致
```

完成后，作品就从“软件状态切换”升级为“真实低压负载通断”。

## 4. 安全底线

必须遵守：

```text
1. 只接 5V/12V 低压负载。
2. 不接 220V 市电。
3. 不接真实电动车电池。
4. 不让 RK2206 直接给小风扇、小电机等负载供电。
5. 负载电源和继电器电源先单独测试。
6. 继电器信号线接入 RK2206 前，先确认触发电平。
7. 上电前检查 VCC/GND，避免反接。
8. 任何异常发热、异味、开发板重启，立即断电。
```

## 5. 继电器模块认识

常见 4 路继电器模块一般有两侧：

### 5.1 控制侧

常见标识：

```text
VCC
GND
IN1
IN2
IN3
IN4
JD-VCC（部分模块有）
```

作用：

- `VCC`：继电器模块逻辑电源。
- `GND`：继电器模块地。
- `IN1`：第 1 路继电器控制信号。
- `IN2/IN3/IN4`：第 2/3/4 路控制信号。
- `JD-VCC`：部分光耦隔离模块的继电器线圈电源。

### 5.2 负载侧

每一路通常有：

```text
COM
NO
NC
```

含义：

- `COM`：公共端。
- `NO`：常开端，继电器动作后导通。
- `NC`：常闭端，继电器未动作时导通。

本项目优先使用：

```text
COM + NO
```

原因：

```text
上电默认断开，只有充电状态才闭合，更符合安全充电逻辑。
```

## 6. 推荐接线方案：第 1 路继电器控制低压负载

### 6.1 总体接线图

```text
RK2206 顶排 B2(GPIO0_PB2) ----------- IN1
RK2206 GND       -------------------- 继电器 GND

独立 5V 电源 +  ---- 急停/保险丝 ---- COM1
NO1              -------------------- 低压负载 +
低压负载 -       -------------------- 独立 5V 电源 -

继电器 VCC       -------------------- 独立 5V 电源 +
继电器 GND       -------------------- 独立 5V 电源 -
```

如果继电器模块需要共地：

```text
RK2206 GND
继电器 GND
独立 5V 电源 -
三者需要共地
```

注意：

```text
是否需要共地取决于继电器模块输入侧设计。
大多数普通 5V 继电器模块需要 RK2206 GND 与继电器 GND 共地，否则 IN1 信号没有参考电平。
如果是真正光耦隔离且 JD-VCC 分离接法，需要按模块说明接线。
```

### 6.2 最小验证接线

为了降低风险，第一次只接：

```text
RK2206 顶排 B2(GPIO0_PB2) -> IN1
RK2206 GND -> 继电器 GND
继电器 VCC -> 5V
继电器 GND -> 5V GND
```

先不接负载。

观察：

- 上电后继电器是否默认不吸合。
- 按 UP 后继电器是否吸合。
- 按 RIGHT 后继电器是否释放。
- 按 DOWN 后继电器是否保持释放。

确认继电器动作后，再接负载。

## 8. 上电前检查表

上电前必须逐项确认：

- [ ] 负载不是 220V。
- [ ] 负载电压为 5V 或 12V。
- [ ] 继电器模块 VCC/GND 没有反接。
- [ ] RK2206 GND 与继电器 GND 是否按模块要求连接。
- [ ] IN1 没有接到 5V 电源正极。
- [ ] GPIO0_PB2/B2 没有和 LCD/RGB/I2C/UART 冲突。
- [ ] 负载接在 COM + NO，而不是 COM + NC。
- [ ] 急停或总电源开关可用。
- [ ] 桌面上没有裸露金属短接风险。
- [ ] 万用表确认 5V 电源输出正常。

## 10. 硬件测试顺序

### Step 1：空板测试

目标：

```text
不接继电器，只确认现有固件正常运行。
```

检查：

- LCD 能显示。
- 按 UP 进入充电。
- 按 RIGHT 进入断电。
- 按 DOWN 复位。
- 串口有 `[safe_charge] event=...`。

### Step 2：继电器控制侧测试，不接负载

目标：

```text
只验证继电器吸合/释放。
```

接线：

```text
GPIO0_PB2/B2 -> IN1
GND -> GND
5V -> VCC
```

检查：

- 上电默认不吸合。
- 按 UP 后吸合。
- 按 RIGHT 后释放。
- 按 DOWN 后保持释放。

如果按 UP 后继电器动作相反，说明可能是低电平触发模块，后续代码需要反相。

### Step 3：接低压负载

目标：

```text
验证真实低压通断。
```

推荐先用小灯：

```text
5V+ -> 急停/保险丝 -> COM1
NO1 -> 小灯 +
小灯 - -> 5V-
```

检查：

- IDLE：小灯灭。
- CHARGING：小灯亮。
- CUTOFF：小灯灭。
- RESET：小灯保持灭。

### Step 4：串口记录

串口期望看到：

```text
event=START state=CHARGING
event=REMOTE_CUTOFF state=CUTOFF
event=MANUAL_RESET state=IDLE
```

### Step 5：LCD 验证

LCD 期望：

| 操作 | LCD 状态 | 负载 |
|---|---|---|
| 上电 | 待机安全 | 灭 |
| 按 UP | 正在充电 | 亮/转 |
| 按 RIGHT | 断电保护 | 灭 |
| 按 DOWN | 待机安全 | 灭 |

### Step 6：MQTT 验证

如果网络已连接，查看字段：

```text
relayStatus: ON/OFF
chargeState: CHARGING/CUTOFF/IDLE
lastEvent: START/REMOTE_CUTOFF/MANUAL_RESET
```

## 11. 高电平触发和低电平触发判断

很多继电器模块有两种触发方式。

### 11.1 高电平触发

现象：

```text
IN1 = 高电平 -> 继电器吸合
IN1 = 低电平 -> 继电器释放
```

当前代码默认逻辑适合高电平触发：

```c
state ? IOT_GPIO_VALUE1 : IOT_GPIO_VALUE0
```

### 11.2 低电平触发

现象：

```text
IN1 = 低电平 -> 继电器吸合
IN1 = 高电平 -> 继电器释放
```

如果是低电平触发，需要后续把输出反相：

```c
IoTGpioSetOutputVal(SAFE_CHARGE_RELAY_GPIO,
    state ? IOT_GPIO_VALUE0 : IOT_GPIO_VALUE1);
```

第一轮先不要猜，实际用继电器动作判断。

## 12. 常见问题排查

### 12.1 按 UP 后继电器没有动作

检查：

- `SAFE_CHARGE_RELAY_ENABLE_GPIO` 是否还是 0。
- GPIO0_PB2/B2 是否接错。
- 继电器 VCC/GND 是否供电。
- RK2206 GND 和继电器 GND 是否共地。
- 继电器触发方式是否与代码相反。

### 12.2 继电器一直吸合

检查：

- 继电器是否低电平触发。
- IN1 是否悬空。
- 上电默认输出是否需要先置高。
- 是否接到了 NC 而不是 NO。

### 12.3 开发板重启

检查：

- 是否用 RK2206 给负载供电。
- 负载电流是否太大。
- 继电器线圈是否拉低开发板供电。
- 5V 电源是否稳定。
- 是否短路。

### 12.4 小灯不亮但继电器有声音

检查：

- 负载回路有没有经过 COM + NO。
- 小灯正负极是否接反。
- 独立 5V 电源是否打开。
- 急停是否断开。
- 保险丝是否断开。

### 12.5 LCD 状态和负载不一致

检查：

- `relay_set_state()` 是否被状态机调用。
- 继电器触发电平是否反相。
- 实物接的是 NO 还是 NC。

## 14. 本阶段完成后才能做什么

完成单路继电器真实通断后，再进入：

```text
第三步：1 号口 INA226/CJMCU-226 功率检测
```

不要跳过继电器稳定性测试直接接 INA226。

原因：

```text
INA226 的意义是检测真实负载电流。
如果继电器和负载通断还不稳定，功率检测数据就没有可靠场景。
```

## 15. 现场操作建议

建议你按这个顺序操作：

```text
1. 拍照记录当前开发板和继电器模块。
2. 找出继电器 VCC/GND/IN1/COM1/NO1。
3. 用标签贴好线名。
4. 不接负载，先测继电器吸合。
5. 再接小灯，不要先接电机。
6. 小灯稳定后再换小风扇或小电机。
7. 每成功一步拍照，后面可放答辩 PPT。
```

答辩时可以说：

```text
我们先完成了 OpenHarmony 端侧状态机到继电器低压负载的真实闭环。
评委按键或远程命令触发状态变化时，不只是屏幕变化，而是继电器真实切断模拟充电负载。
```

### 16.2 图 1 低电平 5V 4 路继电器如何接

图 1 是低电平触发，控制逻辑应理解为：

```text
IN1 = 低电平  -> 第 1 路继电器吸合 -> 负载通电
IN1 = 高电平  -> 第 1 路继电器释放 -> 负载断电
```

所以软件里的继电器逻辑要做反相：

```text
relay_on  -> GPIO 输出 LOW
relay_off -> GPIO 输出 HIGH
```

建议代码增加一个明确宏：

```c
#define SAFE_CHARGE_RELAY_ACTIVE_LOW 1
```

GPIO 初始化时，必须先让继电器处于关闭状态：

```text
先输出 HIGH，再配置为输出模式，避免上电瞬间误吸合。
```

### 16.3 RK2206 接 5V 低电平继电器的风险点

RK2206 的 GPIO 通常按 3.3V 逻辑使用，而你买的继电器模块是 5V 供电。这里不能只看“低电平触发”，还要防止 5V 从继电器 IN 口反灌到开发板 GPIO。

第一轮可以先做无负载测试，但正式推荐两种接法。

#### 推荐接法 A：模块明确支持 3.3V 控制时直连

只有当商家说明写有类似内容时，才建议直连：

```text
支持 3.3V 单片机控制
支持 STM32/ESP32/Raspberry Pi
3.3V/5V 兼容触发
```

直连接线：

```text
RK2206 顶排 B2(GPIO0_PB2) -> 继电器 IN1
RK2206 GND       -> 继电器 GND
外部 5V +        -> 继电器 VCC
外部 5V -        -> 继电器 GND
```

注意：

```text
继电器 VCC 不接 RK2206 的 3.3V。
继电器线圈和负载不要从 RK2206 取电。
```

#### 推荐接法 B：加三极管/ULN2003 做隔离驱动

如果商家没有明确写支持 3.3V 控制，建议加一个 NPN 三极管或 ULN2003 驱动板。

NPN 三极管接法：

```text
RK2206 顶排 B2(GPIO0_PB2) -> 1k 电阻 -> NPN 基极
NPN 发射极       -> GND
NPN 集电极       -> 继电器 IN1
继电器 VCC       -> 外部 5V +
继电器 GND       -> 外部 5V -，并与 RK2206 GND 共地
```

这个接法的好处：

```text
RK2206 只控制三极管，不直接承受继电器 IN 口可能存在的 5V 上拉。
更适合比赛答辩，因为硬件保护思路更完整。
```

如果使用 ULN2003：

```text
RK2206 顶排 B2(GPIO0_PB2) -> ULN2003 IN1
ULN2003 OUT1     -> 继电器 IN1
ULN2003 GND      -> 公共 GND
继电器 VCC       -> 外部 5V
```

### 16.4 图 2 INA226 后续怎么接

INA226 不负责断电，它负责检测：

```text
电压
电流
功率
是否有负载
是否充满
是否异常过流
```

建议在继电器第 1 路稳定后再接 INA226。

低压演示负载推荐接线：

```text
外部 5V/12V 电源 +
-> 保险丝/自恢复保险丝
-> INA226 VIN+
-> INA226 VIN-
-> 继电器 COM1
-> 继电器 NO1
-> 小灯/小风扇 +

小灯/小风扇 -
-> 外部 5V/12V 电源 -
```

INA226 与 RK2206 通信接线：

```text
INA226 VCC  -> RK2206 3.3V
INA226 GND  -> RK2206 GND，并与负载电源负极共地
INA226 SCL  -> RK2206 I2C SCL
INA226 SDA  -> RK2206 I2C SDA
INA226 ALER -> 暂不接
```

注意：

```text
INA226 VCC 建议接 3.3V，这样 I2C 电平也是 3.3V。
不要让 INA226 的 I2C 上拉到 5V。
不要用 INA226 直接测 220V。
不要测超过模块允许范围的电池电压。
```

### 16.5 代码侧需要同步修改

当前工程里已经有 `power_sensor.c`，但第一阶段盘点时确认它更像是 INA219 驱动逻辑。你买的是 INA226，所以后面要做一次驱动适配。

需要改的内容：

```text
1. 保留 power_sensor 抽象接口。
2. 将底层寄存器读取从 INA219 改成 INA226。
3. 默认 I2C 地址优先按 0x40 测试。
4. 读取 bus voltage、shunt voltage、current、power。
5. 把 current/power 接入 charge_safety_update。
6. 在 MQTT safeCharge 属性里上报 voltage/current/power/energy。
7. 在 LCD 上展示实时电压、电流、功率、充电状态。
```

答辩亮点表达：

```text
作品不是只用继电器做简单断电，而是通过 INA226 对充电口侧电压、电流、功率进行实时采样。
系统可以根据负载电流下降、功率变化、异常过流等信息判断充电过程状态，形成检测、控制、反馈、上报的闭环。
```

### 16.6 图 3 继电器模块如何处理

图 3 选择的是 4 路 5VDC。这里的 5VDC 很可能只说明模块供电或继电器线圈电压是 5V，不一定说明它是低电平触发，也不一定说明它支持 3.3V GPIO 控制。

使用前需要确认三个信息：

```text
1. 是高电平触发还是低电平触发。
2. IN1 输入是否支持 3.3V 控制。
3. 有没有光耦隔离、JD-VCC/VCC 分离供电。
```

如果无法确认：

```text
不要作为第一轮主模块。
优先使用图 1 明确标注“低电平 5V 4 路”的模块。
```

## 17. 临时使用开发板 5V 供电的规则

根据实物图，RK2206 扩展底板上可以看到 `5V`、`GND` 等丝印。第一轮如果只是验证单路继电器吸合，可以先从开发板扩展底板取 5V 做临时供电。

但要明确：

```text
这不是最终的独立负载电源方案。
它只适合单路继电器、无负载或很小负载的临时验证。
```

### 17.1 可以这样临时用

适用场景：

```text
1. 只接 1 路继电器。
2. 只验证继电器是否能吸合、释放。
3. 负载先不接，或只接一个小 LED/小灯珠。
4. 开发板由稳定的 USB 适配器或 DC 电源供电。
```

临时接线：

```text
开发板 5V  -> 继电器 VCC
开发板 GND -> 继电器 GND
GPIO0_PB2/B2 -> 继电器 IN1
```

如果用小 LED 做负载：

```text
开发板 5V
-> 继电器 COM1
-> 继电器 NO1
-> LED 正极
-> 限流电阻
-> 开发板 GND
```

如果 LED 模块已经自带限流电阻，可以按模块标识接；如果是裸 LED，必须串联限流电阻。

### 17.2 不建议这样用

不要从开发板 5V 直接带这些负载：

```text
1. 小风扇。
2. 小电机。
3. 四路继电器同时吸合。
4. 多个灯条。
5. 任何真实充电负载。
6. 任何 12V/24V 负载。
```

原因：

```text
电机、风扇启动电流较大，容易让开发板 5V 掉压。
掉压后可能出现开发板重启、LCD 黑屏、Wi-Fi/MQTT 掉线、继电器误动作。
```

### 17.3 上电前必须测量

接线前先用万用表确认：

```text
1. 找到真正的 5V 和 GND。
2. 5V 对 GND 应约为 5.0V。
3. 不要把 5V 接到 GPIO。
4. 不要把 GND 接错到信号脚。
5. 不确定的排针不要凭丝印猜，先测再接。
```

### 17.4 推荐的第一轮测试顺序

```text
1. 开发板上电，不接继电器。
2. 万用表测开发板 5V/GND。
3. 开发板断电。
4. 接 5V/GND 到继电器 VCC/GND。
5. 暂不接 COM/NO 负载。
6. 接顶排 B2(GPIO0_PB2) 到 IN1。
7. 上电后运行程序，让 GPIO 默认输出 HIGH，继电器应保持断开。
8. 按键或命令触发 relay_on，GPIO 输出 LOW，继电器应吸合。
9. 再触发 relay_off，GPIO 输出 HIGH，继电器应释放。
10. 稳定后再接小 LED 负载。
```

### 17.5 什么时候必须换独立电源

出现以下任意情况，就不要再用开发板 5V 带负载：

```text
1. 继电器吸合时 LCD 闪屏。
2. 开发板重启。
3. 蜂鸣器异常响。
4. Wi-Fi/MQTT 断开。
5. 5V 电压低于 4.8V。
6. 准备接小风扇、小电机或 4 路同时演示。
```

正式演示推荐：

```text
开发板自己供电。
继电器线圈和低压负载使用独立 5V/12V 电源。
两边只做必要共地。
```

## 18. 使用面包板电源模块作为 5V 独立电源

根据新增实物图，你手里有一个常见的面包板电源模块，板上有：

```text
DC 圆口输入
7-12V 输入标识
电源开关
5V/OFF/3.3V 跳帽
5V/GND/3.3V 输出排针
USB-A 接口
```

这个模块可以作为第一轮继电器测试的外部 5V 电源，优先级高于直接从 RK2206 扩展底板取 5V。

### 18.1 推荐用途

可以用它供电：

```text
1. 4 路继电器模块的 VCC/GND。
2. 第 1 路低压 LED 小灯负载。
3. 后续少量 5V 指示灯、插口状态灯。
```

暂时不要用它带：

```text
1. 大功率电机。
2. 大电流风扇。
3. 真实电动车充电器。
4. 220V 交流负载。
5. 长时间满载的 4 路大负载。
```

### 18.2 跳帽设置

使用前把黄色跳帽调到 `5V` 档。

```text
左侧电源轨跳帽：拨到 5V。
右侧电源轨跳帽：拨到 5V。
中间排针使用 5V 和 GND。
```

不要拨到 `3.3V` 后再接继电器，否则 5V 继电器可能不吸合或吸合不稳定。

### 18.3 输入电源

推荐输入：

```text
DC 圆口输入 7-12V。
优先使用 7.5V 或 9V 适配器。
电流建议不低于 1A。
```

如果使用 12V 适配器，要注意模块发热。这个模块通常是线性稳压，输入电压越高、负载电流越大，发热越明显。

上电后必须先测：

```text
5V 对 GND = 约 5.0V
3.3V 对 GND = 约 3.3V
```

确认无误后再接继电器。

### 18.4 继电器接线

推荐接线：

```text
面包板电源模块 5V  -> 继电器 VCC
面包板电源模块 GND -> 继电器 GND
RK2206 GND          -> 面包板电源模块 GND
RK2206 顶排 B2(GPIO0_PB2) -> 继电器 IN1
```

这里 `RK2206 GND` 与电源模块 `GND` 必须共地，否则 `B2(GPIO0_PB2) -> IN1` 的控制信号没有共同参考点，继电器可能不响应或误动作。

### 18.5 小负载接线

第一轮建议用 LED 小灯，不要一开始用小风扇。

```text
面包板电源模块 5V
-> 继电器 COM1
-> 继电器 NO1
-> LED 正极
-> 限流电阻
-> 面包板电源模块 GND
```

如果是成品 5V LED 模块，按模块 `+`、`-` 接即可。

如果是裸 LED，必须串联限流电阻。

### 18.6 与 RK2206 的供电关系

建议：

```text
RK2206 开发板单独用 USB 或自身电源供电。
继电器和 LED 小负载由面包板电源模块供电。
两边只共 GND，不把两个 5V 直接硬连在一起。
```

不要这样接：

```text
面包板 5V -> RK2206 5V
RK2206 5V -> 面包板 5V
两个不同电源的 5V 输出直接并在一起
```

这样可能造成两个电源互相顶电，轻则不稳定，重则损坏电源口或开发板。

### 18.7 与低电平继电器的配合

因为继电器是低电平触发：

```text
B2(GPIO0_PB2) = LOW  -> 继电器吸合 -> LED 亮
B2(GPIO0_PB2) = HIGH -> 继电器释放 -> LED 灭
```

正式代码建议加入：

```c
#define SAFE_CHARGE_RELAY_ACTIVE_LOW 1
```

如果发现 `GPIO HIGH` 时继电器仍然轻微吸合、指示灯微亮，说明 3.3V GPIO 与 5V 低电平触发模块匹配不够好。这时应加入 NPN 三极管或 ULN2003 做隔离驱动。

### 18.8 本模块的验收标准

使用面包板电源模块后，本阶段验收标准为：

```text
1. 继电器 VCC/GND 由面包板电源模块提供。
2. RK2206 与面包板电源模块只共 GND。
3. B2(GPIO0_PB2) 能控制 IN1。
4. 继电器吸合时开发板不重启。
5. LED 负载亮灭稳定。
6. 5V 输出不低于 4.8V。
7. 电源模块没有明显烫手。
```
