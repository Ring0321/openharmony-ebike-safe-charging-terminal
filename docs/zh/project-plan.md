# OpenHarmony 社区电动车安全充电项目计划书

面向嵌入式比赛展示与后期平台化扩展，基于 RK2206 OpenHarmony 综合开发板制定。

更新时间：2026-06-16

## 1. 项目定位

本项目面向社区电动车集中充电场景，目标是设计并实现一套“安全充电终端 + 多传感器检测 + 远程监管平台 + 大模型运维问答”的可扩展原型系统。

项目不是单纯做一个传感器数据显示，也不是只做一个 LCD 界面，而是要形成完整闭环：

```text
用户开始充电
-> 终端实时检测
-> 风险评分判断
-> 声光/语音告警
-> 继电器断电保护
-> 事件上报云端
-> 后台查看与大模型问答
-> 人工或管理员复位
```

初期只做低压安全演示，不接 220V 市电。比赛展示重点是让评委一眼看懂系统正在经历哪一个安全流程，并能看到真实硬件动作。

## 3. 项目总体目标

### 3.1 比赛阶段目标

比赛阶段重点做成一个稳定、直观、可讲清楚的安全充电终端。

必须能展示：

1. 待机安全。
2. 用户开始充电。
3. 充电过程中电机转动，表示设备正在工作。
4. 模拟风险告警。
5. 声光提示风险。
6. 进入断电保护。
7. 继电器断开低压负载。
8. 人工复位后回到待机。
9. 云端可以看到状态变化。

### 3.2 后期扩展目标

后期扩展为社区级充电管理系统：

- 多充电口。
- 多节点采集。
- 用户身份识别。
- 预约计时计费。
- 远程断电和复位。
- 故障追溯。
- 后台大屏。
- 大模型安全运维问答。

## 4. 硬件资源盘点与使用规划

### 4.1 当前主控板：RK2206 综合开发板

定位：主控终端。

用途：

- LCD 显示安全充电流程。
- 按键触发演示。
- WiFi/MQTT 上报数据。
- RGB 灯显示状态。
- 板载电机展示“正在充电”。
- 温湿度参与过温风险判断。
- 气体传感器参与烟雾/可燃气体告警。
- 人体感应用于靠近唤醒。
- 光照传感器用于夜间模式。
- SU-03T 语音模块用于播报。

第一阶段只用这块板就可以继续开发，不需要立刻接其他开发板。

### 4.2 INA226 电流功率检测模块

定位：真实电压、电流、功率检测。

用途：

- 替换当前模拟电流和功率。
- 读取低压负载电压。
- 读取充电电流。
- 计算功率。
- 估算能耗 Wh。
- 判断超流风险。
- 判断充满状态。

购买建议：

- 选择 `INA226 / CJMCU-226`。
- 优先选择带蓝色接线端子的版本。
- 接口为 I2C：`VCC / GND / SDA / SCL`。

注意：

- INA226 只测低压回路。
- 初期不要接 220V。
- 到货后放到第二阶段接入。

### 4.3 4 路 5VDC 光耦隔离继电器模块

定位：真实断电保护执行器。

用途：

- 控制 5V/12V 小负载通断。
- 让“断电保护”从屏幕状态变成真实物理动作。
- 后期扩展 4 个充电口。

购买建议：

- 推荐买 `4路 5VDC 光耦隔离继电器模块`。
- 支持高/低电平触发更好。
- 如果只能选一种，选 `低电平触发 5V 4路`。
- 不建议买 12V 或 24V 线圈版本。
- 不建议只买 1 路，因为后期要做多充电口。

接线原则：

- 继电器模块用独立 5V 供电。
- RK2206 控制继电器 IN 引脚。
- RK2206 和继电器模块 GND 必须共地。
- 继电器只控制小灯、小风扇、小电机等低压负载。

### 4.4 低电平触发蜂鸣器模块

定位：告警声光提示的声音部分。

用途：

- 风险告警时短响。
- 断电保护时长响。
- 语音模块不可用时作为备用报警。

第一阶段可以优先接入，难度低，展示效果明显。

### 4.5 SU-03T 语音模块

定位：比赛展示增强。

用途：

- 播报“开始充电”。
- 播报“风险告警，请注意安全”。
- 播报“已断电保护”。
- 播报“人工复位成功”。

重要说明：

- OpenHarmony 固件只能通过串口发送事件码。
- SU-03T 需要使用配套工具把事件码绑定到具体中文播报词。
- 不能只写固件就自动有语音，必须完成语音模块绑定。

### 4.6 智能门锁板：RFID / 指纹 / 键盘

定位：后期身份识别模块。

用途：

- RFID 刷卡开始充电。
- 指纹识别用户。
- 密码输入启动充电。
- 管理员身份解除故障锁定。

阶段安排：

- 第一阶段不接。
- V2.5 身份识别阶段再接入。

### 4.7 MPU6050 姿态传感器

定位：设备防破坏检测。

用途：

- 检测充电箱被撞击。
- 检测设备倾斜。
- 检测异常震动。
- 触发故障锁定或维护告警。

阶段安排：

- 比赛基础版不急着接。
- 后期作为创新点加入。

### 4.8 OLED 小屏

定位：多端口局部显示。

用途：

- 每个充电口显示独立状态。
- 显示端口号、充电状态、功率、告警。

阶段安排：

- V3 多端口阶段使用。

### 4.9 LZ3863 / 星闪小板

定位：后期子节点。

用途：

- 每个子节点负责一个充电口。
- 主控 RK2206 汇总多个节点数据。
- 扩展成多节点社区充电系统。

阶段安排：

- 当前不接，避免分散精力。
- V3 多节点阶段使用。

### 4.10 ST-Link、USB-TTL、TB6612、电机、舵机

| 硬件 | 当前用途 |
|---|---|
| USB-TTL | 调试串口模块、语音模块 |

## 8. 反馈联动逻辑

### 8.1 电机逻辑

板载电机当前先作为“正在充电”的动态展示，不作为真实散热风扇。

```text
待机：电机停
充电：电机转
告警：电机继续转
断电保护：电机停
复位：电机停
```

这样评委能直接看到“充电中有动态，断电后立即停止”。

### 8.2 RGB 灯逻辑

```text
待机：绿色
充电：蓝色
告警：橙色或红色闪烁
断电保护：红色常亮
故障锁定：红色快闪
```

### 8.3 蜂鸣器逻辑

```text
待机：关闭
充电：关闭
告警：短促响
断电保护：长响一次或间歇响
复位：短响一次
```

### 8.4 语音播报逻辑

语音只能在状态变化时播报一次，不能在主循环里反复播报。

| 事件 ID | 状态 | 播报词 |
|---|---|---|
| 0x01 | IDLE | 待机安全 |
| 0x02 | CHARGING | 开始充电 |
| 0x03 | ALERT | 风险告警，请注意安全 |
| 0x04 | CUTOFF | 已断电保护 |
| 0x05 | RESET | 人工复位成功 |
| 0x06 | FULL | 充满提醒，请及时拔车 |
| 0x07 | LOCKED | 故障锁定，请联系管理员 |

固件建议发送简单事件帧：

```text
AA 55 event_id 55 AA
```

如果 SU-03T 配套工具要求其他协议，则以 SU-03T 工具实际协议为准，固件侧只保留 `voice_play_event(event_id)` 抽象接口。

## 9. 远程平台与大模型方案

### 9.1 智能公元产品类型选择

在智能公元创建产品时选择：

```text
Wi-Fi
```

不要选纯离线产品，也不要把“大模型”作为设备类型。

原因：

- RK2206 是 WiFi 设备。
- MQTT 负责上报和控制。
- 大模型运行在云端、PC 或后台服务，不运行在 RK2206 本地。

### 9.2 MQTT 上报数据

建议上报结构：

```json

  "device_id": "rk2206_charge_01",
  "port_id": "port_1",

  "voltage": 5.0,
  "current": 0.52,
  "power": 2.6,
  "energy_wh": 0.03,
  "temperature": 29.3,

  "relay_state": "ON",
  "motor_state": "ON",
  "alarm_state": "OFF",
```

### 9.4 大模型运维问答

大模型方向可以实现，但不要放在板端运行。

推荐架构：

```text
RK2206 -> MQTT -> 云端/PC 后台 -> 数据库/事件记录 -> 大模型问答
```

大模型可回答：

- 当前充电口安全吗？
- 为什么刚才断电？
- 今天发生了几次告警？
- 哪个传感器触发了风险？
- 是否需要人工巡检？
- 请生成今日安全巡检报告。
- 最近异常是否集中在某个充电口？

大模型输入：

- 实时状态。
- 历史事件。
- 风险评分。
- 故障码。
- 传感器曲线。
- 设备在线状态。
- 充电会话记录。

大模型输出：

- 故障原因解释。
- 安全处置建议。
- 维护优先级。
- 巡检报告。
- 答辩演示说明。
- 异常趋势总结。

## 10. 分阶段开发计划

### 10.1 第一阶段：比赛演示稳定版

目标：

在不等待外部硬件到货的情况下，把现有 RK2206 综合板做成稳定、清楚、可演示的作品。

开发内容：

1. 保留当前 LCD 图片资源。
2. 优化界面文字。
3. 统一状态文案。
4. 修正电机逻辑。
5. 增加蜂鸣器或声光状态预留。
6. 增加语音事件接口。
7. 串口输出关键事件日志。
8. MQTT 字段保持兼容。
9. 固定四步演示流程。

验收标准：

- 上键后进入充电，电机转。
- 左键后进入告警，灯光/声光变化。
- 右键后进入断电保护，电机停，界面红色提示。
- 下键后回到待机安全。
- 状态变化清楚，不需要看串口也能理解。

### 10.2 第二阶段：真实低压断电版

目标：

接入 4 路 5VDC 继电器，让断电保护变成真实硬件动作。

开发内容：

1. 继电器驱动。
2. GPIO 控制继电器。
3. 外接 5V/12V 小负载。
4. 断电保护时切断负载。
5. 复位后才能重新开始。

验收标准：

- 充电状态负载通电。
- 断电保护后负载断开。
- 断电后不能自动恢复。

### 10.3 第三阶段：真实功率检测版

目标：

接入 INA226，替换模拟电流功率。

开发内容：

1. I2C 扫描 INA226。
2. 读取电压。
3. 读取电流。
4. 计算功率。
5. 估算能耗。
6. 判断超流。
7. 判断充满。

验收标准：

- LCD 显示真实电流。
- 负载变化时功率变化。
- 超流能触发告警。
- 电流长期低于阈值进入充满提醒。

### 10.4 第四阶段：身份识别与语音完善版

目标：

把作品从“演示设备”升级成“便民设施终端”。

开发内容：

1. SU-03T 完成事件绑定。
2. RFID/指纹/密码识别用户。
3. 用户授权后才能开始充电。

## 13. 安全约束

必须遵守：

1. 初期只做低压演示。
2. 不接 220V 市电。
3. 继电器只控制小灯、小风扇、小电机等低压负载。
4. INA226 只测低压回路。
5. 外部电源接入前确认电压。
6. RK2206 与外部模块共地前确认接线。
7. 断电保护状态下不能自动恢复。
