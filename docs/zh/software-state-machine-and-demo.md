# OpenHarmony 社区电动车安全充电终端：当前软件状态机与演示逻辑

## 1. 当前演示核心结论

当前项目应把 `K1 继电器` 作为真实充电输出，不再把 RK2206 板载电机当作充电负载。

因此演示时的主线是：

```text
按键/云端/语音命令
-> 充电安全状态机
-> K1 继电器控制充电输出
-> K2 继电器控制散热风扇
-> LCD 显示输出、散热、声光、风险状态
```

## 2. 硬件角色划分

| 硬件 | 当前角色 | 当前软件状态 |
|---|---|---|
| RK2206 | 主控、LCD、按键、传感器、状态机 | 已接入 |
| K1 继电器 | 充电输出开关 | 已由 `B2/GPIO0_PB2` 控制 |
| K2 继电器 | 散热风扇开关 | 已由 `B1/GPIO0_PB1` 控制 |
| 外接小灯/电机/第二个风扇 | 临时模拟充电负载 | 接在 K1 后面时，随 K1 通断 |
| 板载电机 | 原始样例里的演示电机 | 已停用，不再自动转 |
| 散热风扇 | 真实保护散热 | 接在 K2 后面，断电保护时继续运行 |

重要原则：

```text
一个风扇不能同时代表“充电负载”和“散热风扇”。

如果它接在 K1 后面：
它代表充电负载，断电保护时应该停止。

如果它要代表散热风扇：
它应该单独接 K2，断电保护时继续运行。
```

## 3. 按键逻辑

| 板子按键 | 软件命令 | 产品含义 | K1 输出 |
|---|---|---|---|
| K3/上键 | `CHARGE_CMD_ACCESS_AUTH_OK` -> `CHARGE_CMD_START` | 模拟刷卡/指纹/密码授权成功，并开始充电 | K1 吸合 |
| K5 | `CHARGE_CMD_SIM_GAS_WARN` | 模拟燃气/烟雾告警 | K1 暂时保持 |
| K6 | `CHARGE_CMD_RELAY_OFF` | 手动/远程断电保护 | K1 断开 |
| K4 | `CHARGE_CMD_RESET` | 复位回待机 | K1 断开 |

推荐现场演示顺序：

```text
K3/上键 授权成功
-> K1 吸合
-> 外接负载工作
-> RK2206 板载电机同步转动，作为“正在充电输出”的可视化展示

K5 模拟异常
-> 进入告警
-> 屏幕显示风险、声光告警

K6 断电保护
-> K1 断开
-> 外接负载停止
-> RK2206 板载电机停止
-> K2 吸合
-> 散热风扇运行
-> 屏幕显示输出断、散热开、声光开

K4 复位
-> 回到待机
-> K1 断开、K2 断开
-> 输出断、散热关、声光关
```

## 4. 状态机逻辑

| 状态 | 进入条件 | K1 充电输出 | 散热状态 | 声光报警 | 屏幕重点 |
|---|---|---:|---:|---:|---|
| `IDLE` 待机 | 上电或复位 | 断 | 关 | 关 | 待机安全 |
| `CHARGING` 充电中 | 授权成功/开始命令 | 通，板载电机转 | 低风险默认关，高温可开 | 关 | 输出通、电流计量 |
| `ALERT` 告警 | 温度/电流/烟雾达到告警阈值，或 K5 模拟告警 | 通，板载电机继续转 | 开，K2 吸合 | 开 | 风险告警 |
| `CUT_OFF` 断电保护 | K6 远程断电，或风险达到切断阈值 | 断，板载电机停止 | 开，K2 吸合 | 开 | 输出断、保护中 |
| `FULL` 充满 | 电流低于满电阈值并持续 | 断 | 关 | 关 | 充满提醒 |
| `OCCUPIED` 占位 | 充满后仍占用 | 断 | 关 | 开 | 占位提醒 |
| `FAULT` 故障锁定 | 多次保护触发 | 断 | 开，K2 吸合 | 开 | 故障锁定 |

## 5. 屏幕显示逻辑

当前屏幕指标应该理解为：

| 屏幕字段 | 含义 |
|---|---|
| 温度 | RK2206 传感器温度或模拟温度 |
| 电流 | INA/模拟电流 |
| 烟雾 | 烟雾/燃气风险值 |
| 输出 | K1 继电器输出，`通` 表示充电输出接通，`断` 表示已切断 |
| 散热 | K2 继电器状态，`开` 表示散热风扇应运行 |
| 人体 | 人体感应状态，`有` 表示检测到人员靠近 |
| 底部 `A:OK/NO` | 门锁/认证授权状态，`OK` 表示允许启动充电，`NO` 表示未授权或已复位 |

## 6. 下一步硬件扩展建议

为了让作品更像真实充电桩，推荐下一步把硬件分成两路：

```text
K1：充电输出继电器，B2/GPIO0_PB2 -> IN1
K2：散热风扇继电器，B1/GPIO0_PB1 -> IN2
```

接入 K2 后，断电保护时可以做到：

```text
充电输出切断
但散热风扇继续运行
```

这比“断电后所有东西都停”更像安全产品。

## 7. 门锁认证板和 LZ3663 串口板接入

### 7.1 两块板子的分工

| 板子 | 在本项目里的作用 | 是否烧录 RK2206 充电固件 |
|---|---|---|
| 紫色智能门锁板 | 做“用户身份认证终端”，可用按键、RFID、指纹、门锁执行机构来模拟用户授权 | 暂时不烧录当前 `Firmware.img`，它不是充电主控 |
| 黑色 LZ3663 板 | 做串口/无线通信桥，把门锁板的授权结果转发给 RK2206 充电主控 | 暂时不烧录当前 `Firmware.img`，只作为外接通信模块 |
| RK2206 安全充电主板 | 负责屏幕 UI、K1/K2 继电器、人体/可燃气体/温湿度/电流检测、断电保护 | 烧录本项目新固件 |

### 7.2 当前固件新增的串口认证协议

RK2206 主板新增 `access_auth` 模块，默认监听：

```text
UART0_M0
RK2206 RX = GPIO0_PB6 / B6
RK2206 TX = GPIO0_PB7 / B7
波特率 115200
格式 8N1
```

门锁板或 LZ3663 串口板发以下任意文本即可触发动作：

| 串口收到文本 | RK2206 动作 |
|---|---|
| `AUTH_OK` / `OK` / `OPEN` / `CARD_OK` / `FP_OK` / `PASS_OK` / `1` | 授权成功，进入充电流程，K1 吸合 |
| `AUTH_DENY` / `DENY` / `FAIL` / `NO` / `0` | 授权失败，保持待机；如果正在充电则切断 |
| `STOP` / `CUT_OFF` / `CUTOFF` | 远程断电保护，K1 断开，K2 散热开启 |
| `RESET` | 复位回待机，K1/K2 断开 |

### 7.3 具体接线

低压调试阶段只接 5V/3.3V 小负载，不接 220V。

```text
LZ3663 GND  -> RK2206 GND
LZ3663 5V   -> RK2206 5V 或独立 5V 正极
LZ3663 TX   -> RK2206 B6 / GPIO0_PB6 / UART0_RX
LZ3663 RX   -> RK2206 B7 / GPIO0_PB7 / UART0_TX
LZ3663 NC   -> 不接
```

如果门锁板本身能输出串口：

```text
门锁板 GND -> RK2206 GND
门锁板 TX  -> RK2206 B6
门锁板 RX  -> RK2206 B7
门锁板 5V  -> 独立 5V 或 RK2206 5V，注意共地
```

### 7.4 新演示闭环

```text
待机 A:NO
-> 门锁板识别成功，发送 AUTH_OK
-> RK2206 显示 A:OK，K1 吸合，进入充电
-> 可燃气体/烟雾/温度/电流异常
-> 进入告警，K1 保持或准备切断，K2 散热开启
-> 断电保护
-> K1 断开，K2 继续散热，蜂鸣/RGB 告警
-> 下键复位
-> 回到待机 A:NO
```

这个逻辑把“身份认证、防蹭充、充电输出、风险检测、断电保护、散热联动、云端上报”串成一个完整闭环。
